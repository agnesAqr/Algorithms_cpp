### 문제 설명

<p>요세푸스 문제는 다음과 같다.</p>

<p>1번부터 N번까지 N명의 사람이 원을 이루면서 앉아있고, 양의 정수 K(≤ N)가 주어진다. 이제 순서대로 K번째 사람을 제거한다. 한 사람이 제거되면 남은 사람들로 이루어진 원을 따라 이 과정을 계속해 나간다. 이 과정은 N명의 사람이 모두 제거될 때까지 계속된다. 원에서 사람들이 제거되는 순서를 (N, K)-요세푸스 순열이라고 한다. 예를 들어 (7, 3)-요세푸스 순열은 <3, 6, 2, 7, 5, 1, 4>이다.</p>

<p>N과 K가 주어지면 (N, K)-요세푸스 순열을 구하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 N과 K가 빈 칸을 사이에 두고 순서대로 주어진다. (1 ≤ K ≤ N ≤ 5,000)</p>

### 출력 

 <p>예제와 같이 요세푸스 순열을 출력한다.</p>

### Review
> list vs. queue
- 두 방식 모두 시간 복잡도는 $O(N \times K)$로 동일. 하지만 실행 속도에서 queue가 더 빠르다.
  - list : 각 노드가 힙 메모리에 개별적으로 할당되어 있어 메모리 구조가 불연속적이고 이는 캐시 미스 현상을 유발하여 속도를 낮춘다.
  - queue : 데이터를 청크(Chunk)/블록(Block) 단위로 관리하여 하나의 블록 안에는 데이터가 연속적으로 배치되어있다. 따라서 캐시 적중률이 list보다 높아 실제 속도가 더 빠르다.

> 원형 순환 문제를 푸는 전략
 1. 제거 순서 or 마지막 하나?
    - 순서 필요 $\rightarrow$ Simulation (Queue/Vector)
    - 마지막만 필요 $\rightarrow$ DP (수학)
 2. $N$(데이터 개수)과 $K$(점프 크기) 확인
    - $N$이 작다 (5,000 이하) $\rightarrow$ Queue
    - $N$은 적당한데(10만), $K$가 엄청 크다(10억) $\rightarrow$ Vector + Modulo
      (큐로 10억 번 돌리면 시간 초과 발생. 수학으로 점프해야 한다.)
      - 대부분의 모듈러(%) 연산은 0부터 시작하는 인덱스에서 가장 깔끔하게 작동한다.
      - 사람 번호가 1번부터라면, vector에는 0~N-1을 넣고 출력할 때만 +1을 해주는 게 계산 실수를 줄일 수 있다.
