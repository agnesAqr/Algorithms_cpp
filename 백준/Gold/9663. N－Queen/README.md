### 문제 설명

<p>N-Queen 문제는 크기가 N × N인 체스판 위에 퀸 N개를 서로 공격할 수 없게 놓는 문제이다.</p>

<p>N이 주어졌을 때, 퀸을 놓는 방법의 수를 구하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 N이 주어진다. (1 ≤ N < 15)</p>

### 출력 

 <p>첫째 줄에 퀸 N개를 서로 공격할 수 없게 놓는 경우의 수를 출력한다.</p>

### review
- 백트래킹 알고리즘의 성능은 아래 두 가지 핵심 요소에 의해 결정된다.<br>
  **1. 탐색 공간의 최소화**<br>
  **2. 유망성 검사 비용의 최적화**
- N-Queen 문제의 경우, 유망성 검사 비용의 최적화를 통해 코드를 좀 더 최적화할 수 있다.
  - for문을 통해 유망성을 검사하면 O(N)이 걸려 비효율적이다.
  - 유망성 검사를 해야 하는 3가지 조건(열, 좌상향, 우상향)을 저장하는 배열에 미리 매핑하면 검사 비용을 O(1)까지 줄일 수 있다. (Space-Time Trade-off)
  - bool 배열 대신 비트마스킹 기법을 적용하면 메모리 사용량을 최소화하고 연산 속도를 더욱 높일 수 있다.
